Atividade: Atributos

1. Qual é a principal função de um Atributo em um banco de dados?

2. Se a Entidade corresponde a uma Tabela, a que corresponde o Atributo?

3. O que é um Atributo Simples? Dê um exemplo.

4. Diferencie um Atributo Simples de um Atributo Composto.

5. O que torna um atributo uma "Super Chave"?

6. Por que o CPF é um bom exemplo de Chave Alternativa?

7. O que é um Atributo Multivalorado? Dê um exemplo.

8. Qual a regra de conversão de um Atributo Multivalorado para o modelo do banco de dados relacional?

9. Na entidade `Carro`, quais seriam alguns atributos? Cite pelo menos três.

10. Por que é importante pensar nos atributos de uma entidade durante a fase de modelagem, e não só na hora de escrever o código?

1. Resposta

Um Atributo é uma propriedade ou característica que descreve uma entidade. Sua função principal é armazenar os dados que dão significado a essa entidade. Por exemplo, na entidade `Cliente`, o atributo `nome` armazena o nome de cada cliente.

2. Resposta

O Atributo corresponde a uma **Coluna** da tabela.

3. Resposta

É um atributo que armazena um único valor e não pode ser dividido em partes menores. Exemplos incluem `idade`, `salário` ou `gênero`.

4. Resposta

Um **Atributo Simples** não pode ser dividido. Já um **Atributo Composto** é uma representação de um agrupamento de dados que podem ser divididos. Por exemplo, `endereço` (composto) é formado por `rua`, `cidade` e `estado` (simples). No banco de dados, o atributo composto se torna vários atributos simples.

5. Resposta

Uma Super Chave é um atributo (ou um conjunto de atributos) que define uma linha de uma entidade como **única**. Em outras palavras, ela garante que não haverá dois registros iguais na tabela.

6. Resposta

O CPF é um bom exemplo porque também é único, mas em muitos cenários práticos, não é a melhor escolha para ser a Chave Primária por conta de possíveis regras de negócio ou performance. Ele pode ser usado para identificar um registro de forma única, mas não será a chave principal de relacionamento.

7. Resposta

É um atributo que pode ter vários valores para uma única entidade. Um exemplo clássico é o `telefone` de uma pessoa, que pode ter mais de um número.

8. Resposta

Um atributo multivalorado, quando mapeado, se torna uma **nova tabela**. Essa nova tabela armazena os valores do atributo e se relaciona com a tabela original.

9. Resposta

`modelo`, `cor` e `ano`.

10. Resposta

A fase de modelagem garante que você identifique todos os dados necessários e defina a estrutura correta para eles, evitando a necessidade de alterações na estrutura do banco de dados (o que pode ser complexo e demorado) após o projeto já estar em andamento. É a etapa de planejamento que assegura a integridade e a qualidade dos dados.
